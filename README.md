# 한글(.hwp) 문서에 스타일을 자동 적용하는 방법

한글 문서에서 제목, 본문, 글머리표, 표 내용의 서식을 일일이 지정하는 작업은 생각보다 시간이 많이 든다.

특히 긴 보고서나 제안서는 내용 작성이 끝난 뒤에도 다음 작업이 반복된다.

- 제목 단계에 맞는 스타일 지정
- 본문과 세부항목 구분
- 표 제목과 표 내용 정렬
- 글꼴, 크기, 줄 간격 재설정
- 문단 번호와 글머리표 수정

이 작업을 줄이기 위해 `.sty` 스타일 파일, 문단 앞의 스타일 태그, 한글 스크립트 매크로를 연결하는 방식을 만들었다.

최종적인 흐름은 간단하다.

> 한글 스타일 저장 → 스타일 분석 → 사용할 스타일 선택 → 태그가 포함된 문서 작성 → 매크로 생성 → 한글에서 자동 적용

![전체 작업 흐름](../assets/images/hwp-style-automation/00-workflow.png)

---

## 1. 작업 결과

문서의 각 문단 앞에 다음과 같은 태그를 붙인다.

```text
[개요1] 가장 맛있는 돼지고기 김치찌개
[본문강조] 좋은 김치찌개는 강한 조미료로 만드는 음식이 아니다.
[본문] 핵심은 돼지고기의 지방에 김치를 충분히 볶는 것이다.
[개요2] 조리 기준
[본문1] 기준 인원은 성인 3~4인이다.
```

한글에서 매크로를 실행하면 다음 작업이 자동으로 처리된다.

1. `[개요1]`, `[본문]` 등의 태그 검색
2. 태그와 뒤쪽 공백 삭제
3. 태그가 있던 문단의 시작으로 이동
4. 연결된 한글 스타일 적용
5. 문서에 남은 같은 태그를 반복 처리

따라서 내용 작성 단계에서는 문단의 역할만 태그로 지정하고, 실제 글꼴과 문단 서식은 한글에서 한 번에 적용할 수 있다.

---

## 2. 배포 파일 구성

예제 압축파일은 다음과 같이 구성했다.

```text
한글_스타일_자동적용_교안_및_예제파일
├─ [교안]한글 스타일 파일 기반 문서 작성 및 자동 스타일 적용.pdf
└─ 예시 파일
   ├─ 1. 준비 파일
   │  └─ [0단계]제안서 틀.hwp
   └─ 2. 산출 파일
      ├─ [0단계]문서스타일.sty
      ├─ [3단계][태그 O]맛있는 김치찌개 레시피.txt
      ├─ [3단계][태그 X]맛있는 김치찌개 레시피.txt
      ├─ [3단계][태그 O]예시 제안서.docx
      ├─ [3단계][태그 X]예시 제안서.docx
      └─ [4단계][코드]스타일 자동 변경 매크로.txt
```

- **교안 PDF:** 전체 작업 순서와 프롬프트, 한글 실행 화면 수록
- **제안서 틀:** 스타일을 추출하기 위한 원본 한글 문서
- **문서스타일.sty:** 한글에서 내보낸 스타일 파일
- **태그 O 파일:** 매크로 실행 전 문서
- **태그 X 파일:** 태그 삭제 및 스타일 적용 후 확인용 문서
- **매크로 코드:** 한글 스크립트 편집기에 붙여넣을 코드

---

## 3. 0단계: 한글 스타일 파일 만들기

먼저 스타일이 등록된 한글 문서를 준비한다.

한글에서 `F6`을 누르거나 다음 메뉴로 이동한다.

```text
서식 → 스타일
```

스타일 가져오기·내보내기 기능을 이용해 새 `.sty` 파일을 만들고, 현재 문서의 스타일을 복사한다.

예시 파일명은 다음과 같다.

```text
문서스타일.sty
```

![한글 스타일 창](../assets/images/hwp-style-automation/01-style-window.png)

이 단계에서 중요한 것은 실제 문서에 사용할 스타일을 미리 정리하는 것이다.

예를 들면 다음과 같다.

| 스타일명 | 용도 |
|---|---|
| 개요1 | 최상위 제목 |
| 개요2 | 중간 제목 |
| 개요3 | 하위 제목 |
| 본문 | 일반 설명 |
| 본문1 | 세부항목 |
| 본문강조 | 핵심 문장 |
| 표중앙 | 표 안의 가운데 정렬 내용 |
| 표왼쪽 | 표 안의 왼쪽 정렬 항목 |
| 표내용 | 표 안의 세부 내용 |

---

## 4. 1단계: `.sty` 파일 분석하기

`.sty` 파일을 AI에 첨부하고 다음 항목을 정리하도록 요청한다.

- 스타일 내부 번호
- 스타일명
- 글꼴과 크기
- 굵게, 기울임, 밑줄
- 문단 정렬
- 들여쓰기와 여백
- 줄 간격
- 문단 위·아래 간격
- 문단 번호 또는 글머리표
- 기타 확인 가능한 서식

여기서 가장 중요한 원칙은 **확인되지 않은 정보를 추측하지 않는 것**이다.

`.sty` 파일의 구조를 완전히 읽지 못했거나 내부 번호가 불명확하다면 다음과 같이 구분해야 한다.

```text
확인 완료
확인 필요
적용 테스트 필요
확인 불가
```

스타일 목록에 보이는 순서를 그대로 내부 번호라고 단정하면 안 된다. 실제 스타일 적용에 사용하는 번호는 문서와 스타일 파일을 기준으로 확인해야 한다.

---

## 5. 2단계: 사용할 스타일 선택하기

분석된 스타일 전체를 모두 사용할 필요는 없다.

문서 작성에 필요한 스타일명만 골라 다음과 같이 정리한다.

| 태그 | 내부 번호 | 스타일명 | 적용 용도 |
|---|---:|---|---|
| `[개요1]` | 2 | 개요1 | 최상위 제목 |
| `[개요2]` | 3 | 개요2 | 중간 제목 |
| `[개요3]` | 4 | 개요3 | 하위 제목 |
| `[개요4]` | 5 | 개요4 | 세부 제목 |
| `[본문]` | 10 | 본문 | 일반 본문 |
| `[본문1]` | 11 | 본문1 | 세부항목 |
| `[본문강조]` | 12 | 본문강조 | 강조 문단 |
| `[주의참고]` | 13 | 주의 및 참고문헌 | 주의사항 |
| `[표중앙]` | 14 | 표중앙 | 표 가운데 정렬 |
| `[표왼쪽]` | 15 | 표왼쪽 | 표 왼쪽 항목 |
| `[표내용]` | 16 | 표내용 | 표 세부 내용 |

내부 번호는 예제 파일 기준이다. 다른 문서나 다른 `.sty` 파일에서는 번호가 달라질 수 있으므로 그대로 복사해서 사용하면 안 된다.

---

## 6. 3단계: 스타일 태그가 포함된 문서 작성하기

문서 작성 단계에서는 각 문단 앞에 `[스타일명] ` 형식의 태그를 붙인다.

태그 뒤에는 반드시 공백 한 칸을 둔다.

```text
[개요1] 문서 제목
[본문강조] 문서의 핵심 내용을 한 문장으로 정리한다.
[개요2] 첫 번째 항목
[본문] 일반적인 설명을 작성한다.
[본문1] 세부항목을 작성한다.
```

스타일에 문단 번호나 글머리표가 포함되어 있다면 번호와 기호를 직접 입력하지 않는다.

잘못된 예시는 다음과 같다.

```text
[개요2] 1. 첫 번째 항목
[본문1] - 세부항목
```

스타일 자체에서 번호와 글머리표를 적용하므로 문서에는 내용만 입력한다.

올바른 예시는 다음과 같다.

```text
[개요2] 첫 번째 항목
[본문1] 세부항목
```

![태그가 포함된 문서](../assets/images/hwp-style-automation/02-tagged-document.png)

기존 문서를 수정할 때도 같은 방식으로 제목과 본문의 위계를 재구성한 뒤 태그를 붙일 수 있다.

## 7. 4단계: 한글 스크립트 매크로 만들기

매크로의 핵심은 태그와 스타일 내부 번호를 연결하는 `styleMap`이다.

```javascript
var styleMap = [
    { tag: "[개요1] ", styleId: 2 },
    { tag: "[개요2] ", styleId: 3 },
    { tag: "[개요3] ", styleId: 4 },
    { tag: "[개요4] ", styleId: 5 },
    { tag: "[본문] ", styleId: 10 },
    { tag: "[본문1] ", styleId: 11 },
    { tag: "[본문강조] ", styleId: 12 },
    { tag: "[주의참고] ", styleId: 13 },
    { tag: "[표중앙] ", styleId: 14 },
    { tag: "[표왼쪽] ", styleId: 15 },
    { tag: "[표내용] ", styleId: 16 }
];
```

태그 문자열 끝에는 공백 한 칸이 포함되어야 한다.

스타일은 다음 방식으로 적용한다.

```javascript
function ApplyRealStyle(styleId)
{
    var action;
    var parameterSet;

    action = CreateAction("Style");
    parameterSet = CreateSet("Style");

    action.GetDefault(parameterSet);
    parameterSet.SetItem("Apply", styleId);
    action.Execute(parameterSet);
}
```

예제 코드에서는 같은 스타일을 두 번 적용한다.

```javascript
ApplyRealStyle(targetStyle);
HAction.Run("MoveParaBegin");
ApplyRealStyle(targetStyle);
```

일부 환경에서 첫 번째 적용만으로 글꼴이나 문단 모양이 완전히 반영되지 않는 현상을 보완하기 위한 방식이다.

전체 매크로 코드는 배포 파일의 다음 문서에 포함했다.

```text
[4단계][코드]스타일 자동 변경 매크로.txt
```

## 8. 5단계: 한글에서 스타일과 매크로 적용하기

### 8.1 스타일 가져오기

한글에서 스타일 창을 열고 필요한 `.sty` 파일의 스타일을 가져온다.

```text
편집 → 스타일 → 스타일(F6)
```

기존 문서의 스타일 구성과 가져올 스타일의 내부 번호가 충돌하지 않도록 적용 전 문서를 별도로 저장하는 편이 안전하다.

![스타일 가져오기](../assets/images/hwp-style-automation/03-import-style.png)

### 8.2 스크립트 매크로 생성

다음 메뉴에서 새 매크로를 만든다.

```text
도구 → 매크로 정의
```

시작 함수명은 다음과 같이 사용한다.

```javascript
function OnScriptMacro_Style()
```

매크로 기록을 중지한 뒤 스크립트 편집기에서 생성한 코드를 붙여넣는다.

![스크립트 편집기](../assets/images/hwp-style-automation/04-script-editor.png)

### 8.3 매크로 실행

코드를 저장하고 실행하면 태그가 순서대로 삭제되면서 각 문단에 스타일이 적용된다.

실행 전에는 원본 파일을 복사해 두는 것이 좋다.

실행 후에는 다음 항목을 확인한다.

- 태그가 모두 삭제되었는지
- 제목 단계가 올바르게 적용되었는지
- 본문 글꼴과 줄 간격이 반영되었는지
- 글머리표와 문단 번호가 중복되지 않았는지
- 표 안의 정렬과 들여쓰기가 유지되었는지
- 일부 문단에 잘못된 스타일이 적용되지 않았는지

## 9. 주의할 점

### 9.1 내부 번호를 임의로 추측하지 않기

스타일명과 내부 번호는 별개의 정보다.

같은 이름의 스타일이라도 문서에 따라 내부 번호가 다를 수 있으므로, 예제의 `styleId`를 다른 문서에 그대로 적용하면 잘못된 스타일이 지정될 수 있다.

### 9.2 태그 뒤의 공백 유지하기

매크로는 `[본문] `처럼 태그와 공백 한 칸을 함께 검색한다.

공백이 빠지거나 두 칸 이상 들어가면 검색되지 않을 수 있다.

### 9.3 먼저 복사본에서 실행하기

매크로는 문서 전체에서 태그를 검색하고 삭제한다.

코드와 내부 번호를 충분히 확인하기 전에는 원본이 아닌 복사본에서 실행해야 한다.

### 9.4 매크로 보안 설정 주의하기

보안 수준을 낮춰야 실행되는 환경에서는 신뢰할 수 있는 코드만 사용해야 한다.

작업이 끝난 뒤에는 매크로 보안 설정을 원래 수준으로 복원하는 것이 안전하다.

### 9.5 적용 결과를 직접 검수하기

자동화는 반복 작업을 줄이는 도구이지 최종 검수를 대신하지 않는다.

특히 표, 문단 번호, 복사한 워드 문서, 개체가 포함된 문서는 별도 확인이 필요하다.

## 10. 활용 가능한 문서

이 방식은 다음과 같은 문서에 활용하기 좋다.

- 연구보고서
- 공공기관 제안서
- 사업계획서
- 용역 수행계획서
- 교육 교안
- 회의자료
- 정기적으로 같은 서식을 사용하는 내부 문서

문서마다 내용은 달라도 사용하는 제목, 본문, 세부항목, 표 스타일이 일정하다면 반복되는 편집 시간을 크게 줄일 수 있다.

## 마무리

이 방식의 핵심은 AI가 한글 서식을 직접 만드는 것이 아니다.

AI는 문서의 구조를 분석하고 각 문단에 스타일 태그를 붙이는 역할을 담당한다. 실제 한글 스타일 적용은 `.sty` 파일과 스크립트 매크로가 처리한다.

역할을 분리하면 다음과 같은 장점이 있다.

- 문서 내용과 서식 작업 분리
- 반복적인 스타일 지정 감소
- 제목과 본문 체계의 일관성 확보
- 같은 스타일을 여러 문서에 재사용
- 수정된 문서에도 동일한 작업 흐름 적용

아직 모든 한글 문서 구조를 완벽하게 자동화하는 방식은 아니지만, 정형화된 보고서와 제안서 작업에서는 충분히 실용적으로 사용할 수 있다.
