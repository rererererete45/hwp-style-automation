# HWP Style Automation

한글 문서에 등록된 스타일을 `.sty` 파일로 내보내고, 선택한 스타일에 맞춰 문서를 작성한 뒤, 문단 앞의 태그를 기준으로 스타일을 자동 적용하는 작업 흐름을 정리한 저장소입니다.

## 목적

한글 문서를 작성할 때 다음 작업을 반복하지 않기 위한 자동화 방식입니다.

1. 기존 한글 문서에서 스타일 파일 내보내기
2. `.sty` 파일에 포함된 스타일 확인
3. 실제 사용할 스타일 선택
4. 선택한 스타일의 위계에 맞춰 문서 작성
5. 각 문단 앞에 스타일 태그 삽입
6. 한글 스크립트 매크로 실행
7. 태그 삭제 및 문단 스타일 자동 적용

## 핵심 원리

문서 작성 단계에서 각 문단 앞에 다음과 같은 태그를 붙입니다.

[개요 1] 최상위 주제

[개요 2] 하위 주제

[본문] 일반적인 설명 내용을 작성하는 문단이다.

[본문 1 (-)] 세부 항목 예시

[본문강조 (○)] 반드시 강조해야 하는 핵심 내용을 작성한다.

[주의 및 참고문헌] 주의사항 또는 참고문헌 정보를 작성한다.

스크립트 매크로는 태그를 검색하여 다음 순서로 처리합니다.

```text
태그 검색
→ 태그 삭제
→ 문단 시작으로 이동
→ 해당 스타일 적용
→ 같은 스타일을 한 번 더 적용
→ 다음 태그 검색
```

스타일을 두 번 적용하는 이유는 일부 한글 환경에서 첫 번째 적용 시 문단 모양만 변경되고 글꼴이나 글자 크기가 완전히 반영되지 않는 현상이 있었기 때문입니다.

# 전체 작업 흐름

## 0단계. 한글 스타일 파일 만들기

사용하려는 스타일이 등록된 기존 한글 문서를 엽니다.

```text
서식
→ 스타일
```

또는 다음 단축키를 사용합니다.

```text
F6
```

스타일 내보내기 기능을 이용하여 현재 문서의 스타일을 `.sty` 파일로 저장합니다.

가능하면 스타일을 일부만 선택하지 말고 전체 스타일을 먼저 내보낸 뒤, 분석 후 사용할 스타일을 선택하는 방식을 권장합니다.

예시 파일명:

```text
제안서_스타일.sty
연구보고서_스타일.sty
공공보고서_스타일.sty
```

## 1단계. `.sty` 파일 분석

생성된 `.sty` 파일을 AI에 제공하고 다음 정보를 정리합니다.

* 내부 번호
* 스타일명
* 글꼴
* 글자 크기
* 굵게 여부
* 정렬
* 들여쓰기
* 줄 간격
* 문단 간격
* 문단 번호
* 글머리표
* 확인 가능한 기타 서식

확인되지 않는 정보는 추측하지 않고 `확인 불가` 또는 `확인 필요`로 표시합니다.

## 2단계. 사용할 스타일 선택

전체 스타일 중 실제 문서에서 사용할 스타일만 선택합니다.

예시:

| 내부 번호 | 스타일명      | 용도       |
| ----- | --------- | -------- |
| 0     | 바탕글       | 부제 또는 설명 |
| 1     | 간지 제목     | 문서 제목    |
| 2     | 개요 1      | 최상위 제목   |
| 3     | 개요 2      | 2단계 제목   |
| 4     | 개요 3      | 3단계 제목   |
| 5     | 개요 4      | 4단계 제목   |
| 6     | 개요 5      | 5단계 제목   |
| 10    | 본문        | 일반 설명    |
| 11    | 본문 1      | 세부 나열    |
| 12    | 본문강조      | 핵심 내용 강조 |
| 13    | 주의 및 참고문헌 | 주의사항과 출처 |

사용하지 않는 스타일은 문서와 스크립트에서 모두 제외합니다.

## 3단계. 스타일 사용 원칙 결정

스타일은 서식을 보여주기 위해 억지로 사용하지 않고 문서의 실제 위계에 따라 사용합니다.

### 개요 스타일

```text
개요 1
└─ 개요 2
   └─ 개요 3
      └─ 개요 4
         └─ 개요 5
```

중간 단계를 특별한 이유 없이 건너뛰지 않습니다.

### 본문

문서의 대부분을 차지하는 일반 설명 문단에 사용합니다.

```text
[본문] 일반 설명
```

### 본문 1

실제로 나열이 필요한 경우에만 제한적으로 사용합니다.

```text
[본문 1 (-)] 첫 번째 세부항목
[본문 1 (-)] 두 번째 세부항목
```

일반 설명 문장을 짧아 보이게 만들기 위한 용도로 남발하지 않습니다.

### 본문강조

정의, 핵심 원칙, 결론 등 반드시 강조할 내용에만 드물게 사용합니다.

```text
[본문강조 (○)] 핵심 명제
```

### 주의 및 참고문헌

다음 내용에만 사용합니다.

* 주의사항
* 해석상의 한계
* 참고문헌
* 출처
* 부연 설명

## 4단계. 태그가 포함된 문서 작성

각 문단의 맨 앞에 적용할 스타일명을 태그로 입력합니다.

태그 작성 규칙:

* 태그는 문단 맨 앞에 배치
* 닫는 대괄호 뒤에 공백 한 칸 입력
* 태그와 내용은 같은 문단에 입력
* 한 문단에는 태그 하나만 사용
* 태그명은 스크립트의 `styleMap`과 완전히 동일하게 작성
* 개요 번호는 직접 입력하지 않음
* 번호는 한글 스타일에서 자동 생성

잘못된 예:

```text
[개요 1] Ⅰ. 변증법적 유물론
```

올바른 예:

```text
[개요 1] 변증법적 유물론
```

## 5단계. 스크립트 매크로 작성

`macro-template.js`의 `styleMap`에 선택한 스타일만 등록합니다.

```javascript
var styleMap = [
    { tag: "[간지 제목] ", styleId: 1 },
    { tag: "[개요 1] ", styleId: 2 },
    { tag: "[개요 2] ", styleId: 3 },
    { tag: "[본문] ", styleId: 10 }
];
```

태그명과 내부 번호는 실제 문서의 스타일 정보에 맞게 수정해야 합니다.

## 6단계. 한글에서 실행

태그가 포함된 문서를 한글에 붙여넣은 뒤 스크립트 매크로를 실행합니다.

실행 전에 원본 문서를 복사하여 테스트용 문서를 만드는 것을 권장합니다.

실행 후 다음 항목을 확인합니다.

* 태그 삭제 여부
* 개요 번호 적용 여부
* 글꼴 적용 여부
* 글자 크기 적용 여부
* 굵기 적용 여부
* 문단 정렬 적용 여부
* 들여쓰기 적용 여부
* 본문 1의 글머리표 적용 여부
* 본문강조의 기호 적용 여부

```text
hwp-style-automation/
├─ README.md
├─ PROMPT.md
├─ macro-template.js
└─ examples/
   └─ dialectical-materialism/
      ├─ style-list.md
      ├─ tagged-document.txt
      ├─ generated-macro.js
      └─ README.md
```

## 파일 설명

### `README.md`

전체 작업 흐름과 사용법을 설명합니다.

### `PROMPT.md`

AI에 `.sty` 파일 분석, 문서 작성, 매크로 생성을 요청할 때 사용하는 프롬프트입니다.

### `macro-template.js`

태그를 찾아 삭제하고 선택한 스타일을 자동 적용하는 기본 스크립트입니다.

### `examples`

실제로 작성한 문서와 생성된 스크립트 사례를 저장합니다.

# 주의사항

* `.sty` 파일의 내부 구조가 항상 완전히 분석되는 것은 아닙니다.
* 스타일 목록 순서와 스크립트 내부 번호가 항상 같다고 단정할 수 없습니다.
* 내부 번호는 실제 적용 결과를 통해 검증해야 합니다.
* 스타일 태그 뒤에는 공백 한 칸이 있어야 합니다.
* 실제 표, 그림, 각주, 미주, 머리말과 쪽 번호는 스타일 적용만으로 생성되지 않습니다.
* 매크로 실행은 반드시 원본이 아닌 복사본에서 먼저 테스트합니다.
* 한글 버전과 환경에 따라 검색 방향값이나 스크립트 동작이 달라질 수 있습니다.
* 이 저장소의 코드는 사용자가 실제로 동작을 확인한 구조를 기준으로 작성되었지만 모든 한글 버전에서의 동작을 보장하지는 않습니다.
