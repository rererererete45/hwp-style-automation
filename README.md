# 한글 스타일 파일 기반 문서 작성 및 자동 스타일 적용

한글에서 내보낸 `.sty` 스타일 파일을 기준으로 문서 구조를 설계하고, AI가 생성한 스타일 태그를 한글 스크립트 매크로로 실제 스타일에 연결하는 **일반형 작업 절차와 프롬프트**를 정리한 저장소입니다.

이 저장소의 중심은 특정 예제 문서가 아니라 다음의 재사용 가능한 흐름입니다.

```text
한글 스타일 파일 생성
→ .sty 파일 분석
→ 사용할 스타일 선택
→ 선택한 스타일로 문서 작성 또는 기존 문서 재구성
→ 태그와 내부 번호를 연결한 매크로 생성
→ 한글에서 스타일 자동 적용
```

예제파일은 위 절차와 프롬프트가 실제로 작동하는지 확인하기 위한 테스트 자료입니다.

## 사용 가이드

전체 교안은 GitHub Pages에서 확인할 수 있습니다.

- [한글 스타일 파일 기반 문서 작성 및 자동 스타일 적용 가이드](https://rererererete45.github.io/hwp-style-automation/)

## 이 저장소에서 제공하는 것

- 한글 `.sty` 스타일 파일 생성 절차
- `.sty` 파일 분석용 프롬프트
- 사용할 스타일 선택용 프롬프트
- 새 문서 작성용 프롬프트
- 기존 문서 재구성용 프롬프트
- 한글 스크립트 매크로 생성용 프롬프트
- 태그 삭제 및 스타일 자동 적용 매크로 구조
- 실제 적용을 확인하기 위한 예제파일

## 단계별 구성

### 0단계. 한글에서 `.sty` 스타일 파일 만들기

스타일이 등록된 한글 문서에서 `.sty` 파일을 내보냅니다.

### 1단계. `.sty` 파일 분석하기

AI가 스타일명, 내부 번호, 글꼴, 크기, 문단 서식, 번호·기호 등을 구분해 정리하도록 프롬프트를 사용합니다.

확인되지 않은 정보는 추측하지 않고 `확인 필요`, `확인 불가`, `적용 테스트 필요`로 구분합니다.

### 2단계. 사용할 스타일 선택하기

분석된 전체 스타일 중 실제 문서에 사용할 스타일만 선택하고, 태그·내부 번호·용도를 연결합니다.

### 3단계. 선택한 스타일로 문서 작성하기

두 가지 방식으로 사용할 수 있습니다.

- 새로운 문서 작성
- 기존 문서 재구성

각 문단 앞에는 다음 형식의 태그를 붙입니다.

```text
[스타일명] 문단 내용
```

스타일에 번호나 글머리표가 포함되어 있다면 번호와 기호는 직접 입력하지 않습니다.

### 4단계. 자동 스타일 적용 코드 만들기

선택한 스타일과 실제 사용된 태그를 기준으로 한글 스크립트 매크로를 생성합니다.

매크로는 다음 작업을 수행합니다.

1. 문서 처음부터 태그 검색
2. 태그와 뒤쪽 공백 삭제
3. 해당 문단의 시작으로 이동
4. 연결된 한글 스타일 적용
5. 동일 태그가 남아 있는 동안 반복
6. 전체 처리 후 문서 처음으로 이동

### 5단계. 한글에서 자동 스타일 적용하기

`.sty` 파일의 스타일을 한글 문서로 가져온 뒤, 생성한 코드를 스크립트 매크로에 붙여넣고 실행합니다.

## 예제파일의 역할

예제파일은 교안의 범용 절차를 설명하기 위한 본문이 아니라, 각 단계의 결과를 확인하기 위한 테스트 자료입니다.

압축파일에는 다음 자료가 포함됩니다.

```text
예시 파일
├─ 1. 준비 파일
│  └─ [0단계]제안서 틀.hwp
└─ 2. 산출 파일
   ├─ [0단계]문서스타일.sty
   ├─ [3단계][태그 O]맛있는 김치찌개 레시피.txt
   ├─ [3단계][태그 X]맛있는 김치찌개 레시피.txt
   ├─ [3단계][태그 O]예시 제안서.docx
   ├─ [3단계][태그 X]예시 제안서.docx
   └─ [4단계][코드]스타일 자동 변경 매크로.txt
```

- `태그 O`: 스타일 태그가 포함된 적용 전 파일
- `태그 X`: 태그를 제거한 적용 결과 확인용 파일
- 매크로 코드: 예제 `.sty` 파일과 예제 문서에 맞춰 생성한 테스트 코드

## 주의사항

- 스타일 내부 번호는 `.sty` 파일과 문서에 따라 달라질 수 있습니다.
- 예제의 `styleId`를 다른 스타일 파일에 그대로 사용하지 마십시오.
- `.sty` 분석 결과에서 확인되지 않은 내부 번호를 임의로 추측하지 마십시오.
- 태그 문자열 끝에는 공백 한 칸이 포함되어야 합니다.
- 매크로는 원본이 아닌 복사본에서 먼저 실행하는 것이 안전합니다.
- 실행 후 글꼴, 문단 번호, 글머리표, 표 정렬을 직접 검수해야 합니다.
- 매크로 보안 수준을 낮춘 경우 작업 후 원래 설정으로 복원하십시오.
