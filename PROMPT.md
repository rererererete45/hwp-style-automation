# HWP 스타일 분석·문서 작성·매크로 생성 프롬프트

한글 문서의 스타일을 분석하고, 사용자가 선택한 스타일에 맞춰 문서를 작성한 뒤 자동 스타일 적용용 스크립트 매크로를 생성한다.

## 기본 작업 순서

```text
0. 기존 한글 문서에서 .sty 파일 내보내기
1. .sty 파일 분석
2. 전체 스타일 목록 정리
3. 사용할 스타일 선택
4. 스타일별 용도와 사용 빈도 결정
5. 태그가 포함된 문서 작성
6. 자동 스타일 적용 매크로 작성
7. 한글 복사본에서 실행 및 검증
```

---

# 0단계. `.sty` 파일 생성 안내

사용자가 아직 `.sty` 파일을 만들지 않았다면 먼저 다음 작업을 안내한다.

```text
한글에서 사용할 스타일이 등록된 문서를 연다.
→ 서식
→ 스타일 또는 F6
→ 스타일 내보내기
→ 현재 문서의 스타일을 새 .sty 파일로 복사
→ 저장
```

가능하면 전체 스타일을 먼저 내보내고, 분석 후 사용할 스타일을 선택한다.

---

# 1단계. `.sty` 파일 분석

사용자가 `.sty` 파일을 제공하면 다음 정보를 표로 정리한다.

| 내부 번호 | 스타일명 | 글꼴 | 크기 | 굵기 | 정렬 | 번호·기호 | 확인 상태 |
| ----: | ---- | -- | -: | -- | -- | ----- | ----- |

확인 대상:

* 내부 번호
* 스타일명
* 문단 스타일 또는 글자 스타일 여부
* 글꼴
* 글자 크기
* 굵게·기울임·밑줄
* 정렬
* 들여쓰기
* 문단 여백
* 줄 간격
* 문단 간격
* 문단 번호
* 글머리표
* 기타 확인 가능한 서식

확인되지 않는 정보는 추측하지 않고 다음과 같이 표시한다.

```text
확인 불가
확인 필요
사용자 제공 정보
실행 결과로 검증 필요
```

파일 분석이 불가능한 경우 다음 자료를 요청한다.

* 스타일 목록 화면
* 스타일 편집 화면
* 글자 모양 화면
* 문단 모양 화면
* 문단 번호 또는 글머리표 화면
* 사용자가 직접 작성한 스타일명과 내부 번호 목록

분석 직후 문서와 코드를 바로 작성하지 않는다. 먼저 사용자가 실제 사용할 스타일을 선택하도록 한다.

---

# 2단계. 사용할 스타일 선택

사용자가 선택한 스타일을 다음 형식으로 다시 정리한다.

| 태그        | 내부 번호 | 스타일명  | 용도     |
| --------- | ----: | ----- | ------ |
| `[간지 제목]` |     1 | 간지 제목 | 문서 제목  |
| `[개요 1]`  |     2 | 개요 1  | 최상위 제목 |
| `[본문]`    |    10 | 본문    | 일반 설명  |

선택되지 않은 스타일은 문서와 스크립트에서 모두 제외한다.

---

# 3단계. 스타일 사용 규칙 확인

사용자가 별도의 원칙을 제시하면 그대로 적용한다.

별도의 원칙이 없으면 다음 기준을 사용한다.

## 개요 스타일

* 개요 1: 최상위 장
* 개요 2: 절
* 개요 3: 항
* 개요 4: 세부항목
* 개요 5: 최하위 세부항목

제목 위계를 특별한 이유 없이 건너뛰지 않는다.

## 본문

일반 설명의 기본 스타일로 사용한다.

문서의 대부분은 본문으로 작성한다.

## 본문 1

실제로 세부 요소를 나열할 때만 제한적으로 사용한다.

```text
[본문 1 (-)] 세부 요소
```

본문 대신 반복적으로 사용하지 않는다.

## 본문강조

핵심 정의, 주요 판단, 결론에만 매우 제한적으로 사용한다.

```text
[본문강조 (○)] 반드시 강조할 내용
```

강조 효과가 약해지지 않도록 남발하지 않는다.

## 주의 및 참고문헌

다음 내용에만 사용한다.

* 주의사항
* 한계
* 출처
* 참고문헌
* 부연 설명

---

# 4단계. 태그가 포함된 문서 작성

각 문단 앞에 다음 형식의 태그를 입력한다.

```text
[스타일명] 문단 내용
```

예시:

```text
[간지 제목] 문서 제목

[개요 1] 최상위 제목

[개요 2] 하위 제목

[본문] 일반 설명 문단

[본문 1 (-)] 세부 나열

[본문강조 (○)] 핵심 강조 문장

[주의 및 참고문헌] 참고문헌 또는 주의사항
```

태그 규칙:

* 문단 맨 앞에 태그 배치
* 태그 뒤 공백 한 칸
* 태그와 내용은 같은 문단
* 한 문단에 태그 하나
* 태그명과 스크립트의 태그명을 완전히 동일하게 작성
* 개요 번호를 직접 입력하지 않음
* 스타일에 포함된 번호와 기호가 자동으로 표시되도록 작성

사용자가 선택하지 않은 스타일은 사용하지 않는다.

스타일을 전부 보여주기 위해 불필요한 제목이나 강조 문단을 만들지 않는다.

---

# 5단계. 스크립트 매크로 작성

문서 작성 후 선택된 스타일만 포함한 완성형 스크립트 매크로를 제공한다.

스크립트 조건:

* 함수명은 `OnScriptMacro_Style`
* 선택된 스타일만 `styleMap`에 등록
* 태그별로 문서 처음부터 검색
* 태그 삭제
* 문단 시작으로 이동
* 실제 한글 스타일 적용
* 같은 스타일을 두 번 적용
* 두 번째 적용 전에 `MoveParaBegin` 실행
* 적용 후 문단 끝으로 이동
* 모든 작업 후 문서 처음으로 이동

반드시 다음 구조를 포함한다.

```javascript
ApplyRealStyle(targetStyle);
HAction.Run("MoveParaBegin");
ApplyRealStyle(targetStyle);
```

한글에서 첫 번째 스타일 적용 시 문단 모양만 변경되고 글꼴이 완전히 반영되지 않는 현상을 보완하기 위한 처리이다.

`styleMap` 예시:

```javascript
var styleMap = [
    { tag: "[간지 제목] ", styleId: 1 },
    { tag: "[개요 1] ", styleId: 2 },
    { tag: "[본문] ", styleId: 10 }
];
```

태그 문자열 끝에는 공백 한 칸을 포함한다.

---

# 6단계. 최종 출력 순서

답변은 다음 순서로 제공한다.

## 1. 적용 기준

* 선택된 스타일
* 스타일별 용도
* 사용하지 않는 스타일
* 본문·나열·강조 스타일 사용 원칙

## 2. 태그가 포함된 문서

복사 가능한 완성 문서로 작성한다.

## 3. 자동 스타일 적용 스크립트

복사 가능한 완성 코드로 작성한다.

## 4. 실행 시 주의사항

* 태그 뒤 공백 한 칸
* 내부 번호 확인
* 복사본에서 먼저 실행
* 검색 방향값이 환경에 따라 달라질 가능성
* 스타일을 두 번 적용
* 표, 각주, 미주, 머리말, 쪽 번호 등은 스타일만으로 실제 개체가 생성되지 않음

---

# 금지사항

* 확인하지 못한 스타일 정보를 추측하지 않는다.
* 사용자가 선택하지 않은 스타일을 문서나 코드에 넣지 않는다.
* 개요의 위계를 무시하지 않는다.
* 본문 1과 본문강조를 남발하지 않는다.
* 실제 표가 없는데 표 스타일을 억지로 사용하지 않는다.
* 실제 그림이 없는데 그림제목이나 캡션 스타일을 사용하지 않는다.
* 스타일 적용과 실제 각주·미주·머리말 등의 기능 생성을 혼동하지 않는다.
* 글꼴과 크기를 직접 지정하는 방식으로 스타일 적용을 대체하지 않는다.
* 실행하지 않은 코드를 검증 완료라고 표현하지 않는다.

---

# 최초 요청 예시

```text
첨부한 한글 .sty 파일을 분석해줘.

아직 글이나 코드는 작성하지 말고 다음 정보만 먼저 정리해줘.

1. 내부 번호
2. 스타일명
3. 글꼴
4. 글자 크기
5. 굵기
6. 문단 정렬
7. 들여쓰기
8. 줄 간격
9. 문단 간격
10. 문단 번호와 글머리표

확인되지 않는 정보는 추측하지 말고 확인 불가라고 표시해줘.

분석이 끝나면 내가 사용할 스타일을 선택할 수 있도록 전체 스타일 목록을 표로 보여줘.
```

# 스타일 선택 이후 요청 예시

```text
다음 스타일만 사용할게.

- 간지 제목
- 개요 1
- 개요 2
- 개요 3
- 개요 4
- 개요 5
- 본문
- 본문 1
- 본문강조
- 주의 및 참고문헌

주제는 변증법적 유물론이야.

본문을 중심으로 작성하고 본문 1은 실제 나열이 필요한 경우에만 사용해줘.
본문강조는 핵심 정의와 결론에만 매우 드물게 사용해줘.

먼저 스타일 태그가 포함된 글을 작성해줘.
그다음 선택한 스타일만 자동 적용하는 한글 스크립트 매크로를 작성해줘.

스타일은 글꼴까지 반영되도록 각 문단에 두 번 적용해줘.
```
